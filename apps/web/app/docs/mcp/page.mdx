# MCP Setup

`caelus-mcp` exposes the engine over the Model Context Protocol. Positions are
computed by the engine on each call, never recalled from training data, so every
answer is deterministic. The server runs over stdio and needs nothing but Node
on the host.

## Run it

The published binary runs straight from npm — no global install:

<CodeBlock lang="bash" label="Terminal" code={`npx -y caelus-mcp`} />

It waits for an MCP client to speak to it over stdio, so an empty terminal is
expected. Wire it into a client below instead of running it by hand.

## Configure your client

Every client points at the same command (`npx -y caelus-mcp`); only the config
file and key differ.

<Tabs>
  <Tab label="Claude Desktop">

Edit `claude_desktop_config.json` (macOS:
`~/Library/Application Support/Claude/`, Windows: `%APPDATA%\Claude\`), then fully
quit and reopen Claude.

<CodeBlock
  lang="json"
  label="claude_desktop_config.json"
  code={`{
  "mcpServers": {
    "caelus": {
      "command": "npx",
      "args": ["-y", "caelus-mcp"]
    }
  }
}`}
/>

  </Tab>
  <Tab label="Claude Code">

Add it from the CLI — it writes the config for you:

<CodeBlock lang="bash" label="Terminal" code={`claude mcp add caelus -- npx -y caelus-mcp`} />

  </Tab>
  <Tab label="Cursor">

Create `.cursor/mcp.json` in the project (or `~/.cursor/mcp.json` for all
projects), then reload Cursor.

<CodeBlock
  lang="json"
  label=".cursor/mcp.json"
  code={`{
  "mcpServers": {
    "caelus": {
      "command": "npx",
      "args": ["-y", "caelus-mcp"]
    }
  }
}`}
/>

  </Tab>
  <Tab label="VS Code">

Create `.vscode/mcp.json` in the workspace — note VS Code uses the `servers`

/>

## Verify the tools appeared

After the client reloads, it should list **seven** tools. Ask it to enumerate
them, or check the client's MCP panel:

`natal_chart`, `current_sky`, `transits`, `synastry`, `find_aspect_dates`,
`rectification_grid`, and `sky_events` (rise/set, phases, stations, zodiac
crossings, solar and lunar eclipses).

If you see all seven, the server is wired correctly. A natal-chart call returns
structured positions, not prose:

<CodeBlock
  lang="json"
  label="natal_chart → (abbreviated)"
  code={`{
  "utc": "1990-06-10T14:30:00Z",
  "houses": "placidus",
  "bodies": {
    "sun":  { "lon": 79.45,  "pos": "19°27'Gem", "house": 11, "speed": 0.96 },

    "saturn": { "lon": 294.31, "pos": "24°18'Cap", "house": 6, "rx": true, "speed": -0.05 }
  }
}`}
/>

## Example prompts

- "Natal chart: born June 10 1990, 2:30pm, Tampa FL."
- "When is Saturn square my natal Moon in the next two years?"
- "Compare my chart with my partner's."
- "Next solar eclipse? Lunar eclipses in 2026?"

## Troubleshooting

- **`npx` not found / server never starts.** GUI clients (Claude Desktop,
  Cursor) do not inherit your shell `PATH`, so a `node`/`npx` installed via nvm
  or Homebrew may be invisible. Use the absolute path as `command` (find it with
  `which npx`), or install Node where the client can see it. Node 18+ is
  required.
- **Tools don't show up.** Most clients only read MCP config on a full restart —
  quit completely (not just close the window) and reopen. Confirm the JSON
  parses (a trailing comma is the usual culprit).
- **First call is slow.** `npx` downloads the package on first run; it is cached
  afterward. Pinning a version (below) avoids surprise re-downloads.

## Version pinning for production

`npx -y caelus-mcp` always resolves the latest release. Pin an exact version so
behaviour is reproducible and the four packages stay in lockstep:

<CodeBlock
  lang="json"
  label="pinned"
  code={`{
  "mcpServers": {
    "caelus": {
      "command": "npx",
      "args": ["-y", "caelus-mcp@0.7.0"]
    }
  }
}`}
/>
